# ES基本查询总结





## 一、先从一个例子开始


```json

{
	"query": {
		"bool": {
			"must": [
				{
					"match": {
						"title": "Search"
					}
				},
				{
					"match": {
						"content": "Elasticsearch"
					}
				}
			],
			"filter": [
				{
					"term": {
						"status": "published"
					}
				},
				{
					"range": {
						"publish_date": {
							"gte": "2015-01-01"
						}
					}
				}
			]
		}
	}
}

```
上面的DSL实现的功能：title中包含”Search”并且content中包含 “Elasticsearch”，status中精确匹配”published”，并且publish_date 大于“2015-01-01”的全部信息。
